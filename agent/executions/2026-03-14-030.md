## Changes made

### `4223398` — send-email EF, Resumen Supabase query, Admin KPIs, password recovery, AdminMensajes, AdminAnalytics

**`supabase/functions/send-email/index.ts`** (NEW):
- Generic transactional email Edge Function via Resend API
- Accepts `{ to, toName, subject, body, trigger, projectId }`
- Converts plain text body to branded HTML email (dark theme, emerald)
- Graceful no-op when RESEND_API_KEY not configured (dev mode)
- Fixes #1008: emailNotifications.ts now has a matching deployed function name

**`src/pages/dashboard/Resumen.tsx`**:
- Added `useEffect` + `useState<ProjectData | null>` for real Supabase query
- Queries `projects` table with `client_id = user.id`, gets unread messages count and pending proposals count in parallel
- Added `ResumenSkeleton` component (animated pulse skeleton for loading state)
- Demo mode: still uses `MOCK_CLIENT_PROJECT` when `isMockDemo()` is true
- Real mode: empty state shows onboarding when no project found

**`src/pages/admin/AdminHome.tsx`**:
- Added `Promise.all` with 3 parallel queries: unread messages, pending payments, total succeeded revenue
- `unreadMessages`: `messages` table where `read_at IS NULL AND sender_role != 'admin'`
- `pendingPayments`: `payments` table where `status = 'pending'`
- `totalRevenue`: sum of `amount` where `status = 'succeeded'`
- Mock fallback: returns realistic counts (3 unread, 2 pending) in mock mode
- Removed TODO comments

**`src/pages/Login.tsx`**:
- Added `mode: 'login' | 'forgot'` state
- Forgot password: calls `supabase.auth.resetPasswordForEmail(email, { redirectTo: origin + '/reset-password' })`
- "¿Olvidaste tu contraseña?" link in password field label row
- Success state with email confirmation message
- Back button to return to login form

**`src/pages/ResetPassword.tsx`** (NEW):
- Route `/reset-password` (added to main.tsx)
- Listens for `PASSWORD_RECOVERY` event via `supabase.auth.onAuthStateChange`
- Form: nueva contraseña + confirmar contraseña (min 8 chars)
- Calls `supabase.auth.updateUser({ password })`
- Success state → redirects to /dashboard after 3s
- Shows warning while session is being established from URL hash

**`src/pages/admin/AdminMensajes.tsx`** (NEW):
- Two-column layout: conversation list (left, 320px) + message thread (right, flex-1)
- Conversation list: avatar initials, client name, project name, last message preview, unread badge, relative time
- Search bar to filter conversations by client name or project name
- Message thread: bubble UI (client = left/zinc, admin = right/emerald), sender avatar, timestamp, unread indicator
- Send message form with paperclip button (future file attach) + text input + send button
- Full mock data: 4 conversations, 9 messages across 4 projects
- Real mode: queries `projects → messages` tables, marks unread on open, sends via INSERT

**`src/pages/admin/AdminAnalytics.tsx`** (NEW):

- Revenue bar chart: last 6 months, animated bars, emerald gradient

- Conversion funnel: stacked bars from leads → questionnaires → proposals → accepted → payments
- Full mock data (leadsThisMonth: 6, conversion: 62%, revenue: 25.299€, MRR: 298€)
