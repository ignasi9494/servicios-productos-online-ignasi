## Backlog Item Implemented
### #1027 — Realtime en AdminMensajes (Severity: MEDIUM)

**What was done:**
1. Added Supabase Realtime subscription via `supabase.channel('admin-messages-realtime')` listening for `postgres_changes` INSERT events on the `messages` table
2. New messages in the currently active conversation are appended to the message list in real-time (no page refresh needed), with duplicate prevention
3. New messages in other conversations update the unread badge count and re-sort the conversation list by recency
4. Browser Notification API integration: permission requested on mount; notification fired when admin receives a message from a client while viewing a different conversation
5. "En vivo" / "Offline" pill badge in the header indicates realtime connection status (emerald = connected, zinc = disconnected)
6. `activeProjectId` tracked via a `useRef` so the async Realtime handler always reads the current conversation ID
7. Channel properly cleaned up on unmount via `supabase.removeChannel(channel)`
8. Removed unused `MOCK_PROJECTS` and `MOCK_CLIENTS` imports (lint cleanup)
