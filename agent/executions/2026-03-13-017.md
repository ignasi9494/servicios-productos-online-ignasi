# Execution #17 — 2026-03-13

## Task
**#302** — Internal chat system (client side)

## What was done
Built the complete client-side real-time chat system for client-team communication within the dashboard.

### ChatBubble component (`src/components/dashboard/ChatBubble.tsx`):
- **Three bubble types**: client messages (emerald, right-aligned), admin messages (grey, left-aligned with name), system messages (centered pill)
- **Avatars**: Initials-based, emerald for own messages, cyan for team messages
- **Read receipts**: Single check (sent), double check in cyan (read)
- **Attachments**: Link to view attached files
- **Timestamps**: HH:MM format below each bubble

### ChatInput component (`src/components/dashboard/ChatInput.tsx`):
- **Text input**: Auto-resizing textarea with Enter to send, Shift+Enter for newline
- **File attachments**: Paperclip button, file preview with remove option
- **Send button**: Emerald when can send, disabled grey otherwise
- **File types**: Accepts images, PDFs, Office documents

### Mensajes page (`src/pages/dashboard/Mensajes.tsx`):
- **Project discovery**: Fetches user's latest project from Supabase
- **Message loading**: Loads all messages with sender name resolution from profiles table
- **Realtime subscriptions**: Supabase Realtime for INSERT and UPDATE events on messages table
- **Auto-read**: Marks incoming messages as read automatically when chat is open
- **File upload**: Uploads attachments to Supabase Storage `chat-files` bucket
- **Auto-scroll**: Scrolls to bottom on new messages
- **Empty states**: Different messages for "no project" vs "no messages yet"
- **Graceful fallback**: Shows placeholder when Supabase not configured
