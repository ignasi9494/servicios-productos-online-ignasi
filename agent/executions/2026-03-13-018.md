## What was done
Built the complete proposal viewer with version history, markdown rendering, contract acceptance flow, and Stripe payment integration.

### Propuestas page (`src/pages/dashboard/Propuestas.tsx`):
- **Version selector**: Horizontal pills showing all proposal versions with status icons (clock=pending, check=accepted, x=rejected)
- **Proposal detail view**: Full proposal content display with:
  - Status banner (amber for pending, emerald for accepted)
  - Markdown content renderer (headings, lists, bold, paragraphs, dividers)
  - Price breakdown table from `price_breakdown_json`
  - Timeline and stack description sections
  - Sent date and version number
- **Acceptance flow**: Three-step process:
  1. Review proposal → click "Aceptar propuesta"
  2. Contract summary showing: project name, plan, version, total price, deposit (40%), final payment (60%)
  3. Checkbox "He leido y acepto los terminos" → "Firmar y proceder al pago"
- **On accept**:
  - Updates proposal status to 'accepted' with `responded_at`
  - Updates project status to 'proposal_accepted' with `contract_signed_at`
  - Creates Stripe Checkout session for 40% deposit payment via `createCheckoutSession()`
  - Redirects to Stripe hosted checkout
- **Request changes**: Navigates to `/dashboard/mensajes` for chat with team
- **Empty state**: CTA to start questionnaire when no proposals exist
- **Graceful fallback**: Works without Supabase configured
