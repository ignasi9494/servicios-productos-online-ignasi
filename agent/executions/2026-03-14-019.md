## Changes made

### #304 - Document manager (`src/pages/dashboard/Documentos.tsx`)
New page at `/dashboard/documentos`:
- **Drag-and-drop upload zone** — handles multiple files, max 20 MB per file, click-to-browse fallback
- **Two sections**: "Tus archivos" (uploaded by current user) and "Archivos del equipo" (uploaded by admin/team)
- **Filter tabs**: Todos / Documentos / Imágenes / Propuestas
- **File row** with icon (by type), name, uploader name (with "(Equipo)" suffix for admin), date, size
- **Hover actions**: eye (image preview), download, delete (own files only)
- **Image preview modal** with AnimatePresence open/close animation
- **Supabase Storage** backend: uploads to `project-files` bucket, metadata in `files` table
- **Graceful empty state** when no project exists

### #305 - Iteration request system (`src/pages/dashboard/Iteraciones.tsx`)
New page at `/dashboard/iteraciones`:
- **Iterations counter card** with color-coded remaining count (emerald/amber/red)
- **Last-iteration warning banner** (amber) when only 1 iteration left
- **Inline request form** (AnimatePresence animated):
  - Description textarea (required)
  - Screenshot upload (multiple images)
  - Submit creates `iterations` row + auto-sends message to team chat
  - Updates `used_iterations` count on project
- **Iteration cards** (collapsible with AnimatePresence):
  - Status badge: Solicitada (amber clock) / En progreso (blue spinner) / Completada (emerald check)
  - Description, screenshots grid (clickable), completion date
- **Buy more CTA** (+250€) when iterations exhausted, links to `/dashboard/mensajes`
- **Graceful empty state** with CTA to start questionnaire

### #602 - Toast system + 404 page
**`src/contexts/ToastContext.tsx`**:
- `ToastProvider` wraps the app in `main.tsx`
- API: `toast()`, `success()`, `error()`, `info()`, `warning()` helpers via `useToast()` hook
- Animated toast stack (bottom-right, max 5 visible, AnimatePresence spring animation)
- Auto-dismiss (4s success/info, 5s warning, 6s error), manual X dismiss
- Distinct styling per type (emerald/red/amber/zinc)

**`src/pages/NotFound.tsx`**:
- Styled 404 page matching dark theme
- Giant gradient "404" text (emerald → cyan)
- "Volver atrás" (history.back()) + "Ir al inicio" (Link to "/")
- Registered as `<Route path="*">` catch-all in main.tsx

### Routing + DashboardLayout changes
