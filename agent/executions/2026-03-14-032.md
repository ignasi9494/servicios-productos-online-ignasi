# Execution #32 — 2026-03-14

## Task
**P0 Critical bug**: Post-login stuck loading (both admin and client users)

## Root Cause
Race condition in `AuthContext.fetchProfile`: `onAuthStateChange` fires and calls `fetchProfile` immediately after login, but the JWT token hasn't propagated to the PostgREST REST API yet. The first `GET /rest/v1/profiles` returns **401 Unauthorized** (PGRST301). The existing retry logic only handled `AbortError`, so on a 401 it returned `null` and `loading` was set to `false` with no profile — causing guards to redirect or show a spinner indefinitely depending on route.

Evidence from Supabase API logs:
- `POST /auth/v1/token → 200` (login ok)
- `GET /rest/v1/profiles → 401` (JWT not yet propagated)
- `GET /rest/v1/profiles → 200` (works 300ms later)

## Fix Applied
**File**: `src/contexts/AuthContext.tsx`, function `fetchProfile` (lines ~88-93)

```ts
// Before
if (attempt < 3 && error.message?.includes('AbortError')) {

// After
const is401 = (error as { status?: number }).status === 401 || error.code === 'PGRST301';
if (attempt < 3 && (error.message?.includes('AbortError') || is401)) {
  await new Promise(r => setTimeout(r, 300 * attempt));
```

Changed delay from 400ms to 300ms per attempt (300ms, 600ms) — faster recovery while still giving enough time for token propagation.
