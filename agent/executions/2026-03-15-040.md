## Backlog Item Implemented
### #1019 — Questionnaire completion → create project in Supabase (unauthenticated users)

**Problem**: When an unauthenticated user completed the questionnaire and saw the price reveal, they could click "Regístrate para recibir tu propuesta" but the price/plan/project data was lost. After registering, they landed on an empty dashboard with no project record in Supabase.

**Solution**:

1. **`src/lib/pendingProject.ts`** (new file):
   - `PendingProjectData` type: projectName, plan, basePrice, extrasTotal, totalPrice, deliveryDays, maxIterations, sessionId
   - `createProjectFromPending(userId)`: reads localStorage, inserts row into `projects` table, links questionnaire session, cleans up localStorage
   - `hasPendingProject()` / `getPendingProject()`: check/read the pending data

2. **`src/components/questionnaire/PriceReveal.tsx`**:
   - Added `savePendingProject()` helper: saves all pricing data + sessionId to `localStorage['tb_pending_project']` before navigating
   - Updated CTA links to `/registro?from=cuestionario` and `/login?from=cuestionario`
   - `onClick` on both links calls `savePendingProject()` before navigation
   - Removed unused `AnimatePresence` import

3. **`src/pages/Login.tsx`**:
   - Detects `?from=cuestionario` URL param and reads `getPendingProject()`
   - Shows project banner (name + plan + price) when pending project exists
   - On successful login: if `hasPendingProject()`, calls `createProjectFromPending(user.id)` before navigating to dashboard
   - Added `FileText` icon import

4. **`src/pages/Registro.tsx`**:
   - Detects `?from=cuestionario` URL param
   - Shows contextual subtitle: "Crea tu cuenta para recibir la propuesta definitiva"
   - Shows pending project banner (name + plan + price) when `tb_pending_project` exists
   - Project creation deferred to first login (after email confirmation) — handled by Login.tsx
   - Added `FileText` icon and `useLocation` imports
