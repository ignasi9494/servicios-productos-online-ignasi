## Bugs Fixed

### createRoot HMR duplicate warning (main.tsx)
- **Symptom**: Console showed repeated `ReactDOMClient.createRoot() on container already passed to createRoot()` errors in dev mode
- **Root cause**: Vite HMR re-evaluates main.tsx on every hot reload, calling `createRoot()` on the same container again
- **Fix**: Cache the root instance on the DOM container as `__reactRoot`; reuse it if present

## Backlog Item Implemented

### #1020 — Storage error handling improvements

**Documentos.tsx**:
- Increased max file size from 20 MB to 50 MB
- Added `isAllowedType()` validation before any upload starts
- Toast notifications on upload success and error (replaces inline error state for per-file errors)
- Upload progress bar (`uploadProgress { current, total }`) — drag zone shows spinner + "X de N archivos..." + animated progress bar during upload
- Friendly "almacenamiento no configurado" message when bucket is missing (instead of raw Supabase error)
- Each file is attempted independently — failures don't abort the whole batch

- Added `CHAT_MAX_FILE_SIZE` (10 MB) constant
- Validates attachment size before uploading; shows toast and returns early if exceeded
- Handles storage upload error with toast (bucket-not-found gets friendly message)
- If upload fails, message still sends without attachment (graceful degradation)

**Iteraciones.tsx**:
- Added `ITER_MAX_FILE_SIZE` (20 MB) and `image/*` type check on file picker onChange
- Oversized files shown in toast; valid files still added to selection
- In handleSubmit: per-file validation (size + image type) before starting upload loop
- If upload fails, iteration still submits without screenshot URLs (graceful degradation)
- Toast distinguishes bucket-not-found vs other upload errors
