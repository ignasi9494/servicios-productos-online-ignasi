**`src/lib/questionnaireEngine.ts`**:
- After getting engine response (Edge Function or fallback), compute `minProgress = Math.min(95, ceil(history.length/2) * 5)`
- If `response.progressPercent` is 0 or less than current progress, use `max(currentProgress, minProgress)` instead
- Ensures bar always advances with each user message, never regresses to 0

**`src/components/questionnaire/ChatUI.tsx`**:
- Root cause of session recovery bug: `pendingSession.messages` was always `undefined` because engine state uses `history` (not `messages`)
- Fix: check `Array.isArray(pendingSession.history)` instead
- Convert `ConversationMessage[]` (engine format: `{role: 'user'|'model', parts: [{text}]}`) to `ChatMessageData[]` (UI format: `{id, role: 'user'|'bot', content, timestamp}`) so previous messages render correctly

**`src/contexts/AuthContext.tsx`**:
- Added: `const mockRole: UserRole = import.meta.env.VITE_MOCK_ROLE === 'admin' ? 'admin' : 'client'`
- mockProfile now uses mockRole, name/company change accordingly

**`.env.example`**:
- Documented `# VITE_MOCK_ROLE=admin` with comment
