## What was done

### Edge Function: `questionnaire-chat`
- Created `supabase/functions/questionnaire-chat/index.ts`
- Uses `@google/genai` SDK to call Gemini 2.5 Flash
- Accepts: `{ sessionId, userMessage, selectedOption?, uploadedFiles?, conversationHistory[], systemPrompt }`
- Returns structured JSON: `{ botMessage, componentToRender?, componentProps?, isComplete, extractedData?, progressPercent }`
- Uses Gemini's `responseMimeType: 'application/json'` with `responseSchema` for structured output
- CORS headers for cross-origin requests
- Requires `GEMINI_API_KEY` env var in Supabase

### Client-side engine: `questionnaireEngine.ts`
- `QuestionnaireEngine` class manages conversation state
- Sends messages to Edge Function via fetch with 30s timeout
- Rate limiting: min 1s between messages
- Fallback mode: when Edge Function unavailable, provides a 10-question static flow with embedded components (CardSelector, MultiSelect, ColorPicker, URLInput, FileUpload, Slider, TextArea)
- Persists state to localStorage (`tb_questionnaire_engine`)
- Fire-and-forget persistence to Supabase `questionnaire_conversations` table
- `engineResponseToChatMessage()` helper converts engine responses to `ChatMessageData`

### ChatUI integration
- Replaced mock `getBotReply()` with real `QuestionnaireEngine`
- `handleComponentComplete` callback handles embedded component results
- `sendToEngine` async function manages typing state, error handling
- Input disabled when `engine.isComplete`
