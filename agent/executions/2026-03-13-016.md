**#301** — Project status tracker component

## What was done
Created a standalone, reusable `ProjectStatus` component and refactored Resumen.tsx to use it.

### ProjectStatus component (`src/components/dashboard/ProjectStatus.tsx`):
- **8 stages** matching DB `project_status` enum: Cuestionario completado → Propuesta enviada → Propuesta aceptada → Pago de entrada → En desarrollo → En revision → Completado → Entregado/Mantenimiento
- **Desktop**: Horizontal timeline with nodes, connector lines, icons per stage
- **Mobile**: Vertical timeline with connector bars (responsive, `sm:hidden` / `hidden sm:block`)
- **Stage states**: Completed (emerald + checkmark), current (cyan + clock + ring), pending (grey + stage icon)
- **Stage dates**: Optional `stageDates` prop shows formatted dates under completed stages
- **Dev progress**: Optional `devProgress` prop shows a mini progress bar on "En desarrollo"
- **Iteration info**: Optional `iterationInfo` prop shows "Iteracion X/Y" on "En revision"
- **Status mapping**: `STATUS_TO_INDEX` maps DB enum values to stage indices
- **Motion animation**: Wraps in motion.div with fade-in by default, configurable via `animate` prop

### Resumen.tsx refactored:
- Replaced inline 7-stage pipeline with `<ProjectStatus status="proposal_sent" />`
- Removed unused `PROJECT_STAGES` array and `currentStage` variable
- Removed unused `CheckCircle2` and `Clock` imports
