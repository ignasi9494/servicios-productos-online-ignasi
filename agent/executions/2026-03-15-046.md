## Backlog Item Implemented

### #1010 — Entrega.tsx: pay-to-export Stripe CTA (HIGH priority)

**Problem**: The export option in `Entrega.tsx` had no payment path. When an admin creates a pending `final`/`full` payment for a completed project, the client had no way to pay and unlock the code download from the Entrega page. They had to go to Pagos → "Pagar ahora" instead.

**Changes to `src/pages/dashboard/Entrega.tsx`**:
1. Added `PendingPayment` interface (`id`, `amount` in cents, `type`)
2. Added `pendingFinalPayment` state and `exportLoading` state
3. In `useEffect`: after fetching project, query `payments` table for any `pending` rows with `type in ('final', 'full')` for the project — stores as `pendingFinalPayment`
4. Added `handleExportPay()` function: triggers real Stripe checkout via `createCheckoutSession` with the pending payment amount, tracks analytics event
5. Updated export detail panel with 3 conditional states:
   - `delivery_url` set → direct download link (unchanged)
   - `pendingFinalPayment` → amber warning banner with amount + "Pagar y descargar código" button
   - No payment, no URL → "Preparando tu entrega" waiting state (unchanged)
6. Export option card hint updated to show pending payment amount (not just clock icon)
7. Mock mode: simulates a 3500€ pending payment so the new flow is visible in demo
