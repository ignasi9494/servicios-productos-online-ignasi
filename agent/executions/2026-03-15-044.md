### #1011 — Admin: preview_url input in Entrega tab

**What was missing:** The client `Preview.tsx` reads `preview_url` from the `projects` table to show the app in an iframe — but the admin had no way to SET this value from the admin panel. The Entrega tab only had a ZIP upload and "Mark as delivered" button.

**What was added to `src/pages/admin/AdminProjectDetail.tsx`:**
1. Added `preview_url: string | null` field to the `Project` interface
2. Added `previewUrl` state (string) and `savingPreviewUrl` boolean state
3. Both mock and real Supabase data loading paths now set `previewUrl` from `proj.preview_url`
4. Added `handleSavePreviewUrl()` — trims input, calls `supabase.from('projects').update({ preview_url })`, shows success/error toast
5. Added "URL de vista previa" card in the Entrega tab:
   - Cyan Eye icon + title
   - Descriptive text explaining it shows in client's iframe
   - URL text input with placeholder `https://mi-proyecto.vercel.app`
   - Guardar button with loading spinner
   - Active preview link indicator (shown when `project.preview_url` exists)
6. Updated delivery flow instructions to include preview URL as step 1

**Result:** Admin can now enter a staging/preview URL (e.g. Vercel preview deploy), save it, and the client immediately sees it in their "Vista previa" dashboard section.
